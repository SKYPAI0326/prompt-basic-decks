## 設計原則（嚴格遵守）

這份 deck 為 **公部門行政人員** 設計，與一般 AI 教學不同：

### 用語規範（術語對照表）

| 禁用 | 替代 |
|---|---|
| Prompt / 提示詞 / 指令工程 | **交辦單 / 交辦** |
| Few-shot | **給範例** |
| Chain-of-Thought / CoT | **請它一步步想** |
| Context / Context window | **背景資料** |
| Token | **字數** |
| Structured output | **指定回答的樣子** |
| Meta-prompt | **問題清單**（必要時才出現） |
| RTCF / CRISPE | 不出現 |

### 情境白名單

只用：陳情案回函 / 會議紀錄整理 / 公文撰寫 / 長文摘要 / 潤稿翻譯 / 向長官呈報。
**不用**：行銷文案、廣告、程式碼、技術文件、學術論文、創意寫作、求職、客服對話。

### 語氣

- 直接，不廢話
- 不誇張（無「神奇」「保證有效」）
- 自曝侷限（每招說明「什麼時候不適用」）
- 禁用行銷話術（「AI 取代人力」「神級 prompt」「終極咒語」全禁）

## 操作方式

| 動作 | 鍵 |
|---|---|
| 翻頁 | `←` `→` / 滾輪 / 觸控滑動 / 底部圓點 |

## 技術規格

- **單檔 HTML** · 不依賴後端，離線可跑
- **WebGL 背景** · Hero 頁的雙 shader（暗色色散 + 亮色流動）
- **Motion One 動效** · cascade / hero / quote / directional / pipeline 五種 recipe
- **Lucide 圖示** · 線性圖標 CDN
- **Google Fonts** · Noto Serif SC + Noto Sans SC + Playfair Display + IBM Plex Mono
- **無框架依賴** · 純 HTML / CSS / 原生 JS
- **手機響應式** · 豎屏提示 overlay + 字型補強
